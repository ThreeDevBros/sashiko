# iOS Setup Guide — Full Native Build

Complete guide to build the iOS app from scratch with **Push Notifications**, **Live Activities**, **Firebase**, and all native plugins.

---

## Prerequisites

- **Mac** with **Xcode 15+** installed (Xcode 26 supported with patch below)
- **iOS 16.2+** physical device (Live Activities & push notifications don't work in Simulator)
- **Apple Developer account** with Push Notifications capability
- **Firebase project** with iOS app configured
- Project exported to GitHub and cloned locally

---

## Step 1: Clean iOS Rebuild

```bash
# From the project root
rm -rf ios

npm install
npm run build
npx cap add ios
npx cap sync ios
```

### Xcode 26 Compatibility Fix

If you're using Xcode 26 (beta or release), patch the project format:

```bash
sed -i '' 's/objectVersion = 70;/objectVersion = 77;/' ios/App/App.xcodeproj/project.pbxproj
```

## Step 2: Firebase Setup

### 2a. Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **Add project** → follow the wizard
3. In your project, click **Add app** → choose **iOS**
4. Enter the bundle ID: `app.lovable.6e0c6b4d4b7943e7a8431d08565d9c10`
5. Download `GoogleService-Info.plist`

### 2b. Add GoogleService-Info.plist to Xcode

1. Open `ios/App/App.xcworkspace` in Xcode
2. Drag `GoogleService-Info.plist` into the **App** folder in the navigator
3. ✅ Check **"Copy items if needed"**
4. ✅ Ensure **Target Membership** includes **App**

### 2c. Add Firebase Pods

Edit `ios/App/Podfile` and add these lines inside the `target 'App'` block:

```ruby
target 'App' do
  capacitor_pods
  # Add these two lines:
  pod 'FirebaseCore', '~> 11.0'
  pod 'FirebaseMessaging', '~> 11.0'
end
```

Then install pods:

```bash
cd ios/App
pod install --repo-update
cd ../..
```

## Step 3: Add Native Swift & Objective-C Files

Copy the files from this `/setup/swift/` folder into `ios/App/App/` using Xcode.

### How to add files in Xcode

1. Open `ios/App/App.xcworkspace` in Xcode
2. Right-click the **App** folder in the navigator → **New File…** (or drag files in)
3. For each `.swift` file: choose **Swift File**, name it accordingly, paste the code
4. For `.m` files: choose **Objective-C File**, name it accordingly, paste the code
5. If Xcode asks to create a **bridging header**, click **Create**

### Files to add to the App target

| File | Description |
|------|-------------|
| `PushNotificationSetup.swift` | Firebase init + FCM token capture + push permission handling. **New file — do NOT replace Capacitor's AppDelegate.** |
| `PushNotificationBridge.m` | Objective-C hook that auto-configures push notifications on launch without touching AppDelegate. |
| `GenericAttributes.swift` | Shared `ActivityAttributes` struct for Live Activities. Must be added to **both** App and Widget targets. |
| `LiveActivityPlugin.swift` | Custom Capacitor plugin for Live Activity lifecycle. |
| `LiveActivityPlugin.m` | Objective-C bridge for the Live Activity plugin. |

### Important: Do NOT replace Capacitor's AppDelegate

Keep Capacitor's default `AppDelegate.swift` as-is. The `PushNotificationBridge.m` file automatically hooks into app launch via `NSNotificationCenter` and calls `PushNotificationSetup.shared.configure()` — no AppDelegate modification needed. This prevents the black screen issue that occurs when Capacitor's scene/window lifecycle is disrupted.

## Step 4: Add Widget Extension (Live Activities)

### 4a. Create the Widget Extension

1. In Xcode: **File → New → Target…**
2. Choose **Widget Extension**
3. Name it `OrderTrackingWidget`
4. ✅ Check **"Include Live Activity"**
5. Click **Finish**

### 4b. Share GenericAttributes.swift with the Widget

1. Select `GenericAttributes.swift` in the file navigator
2. In the **File Inspector** (right panel), under **Target Membership**, check both:
   - ✅ **App**
   - ✅ **OrderTrackingWidgetExtension**

### 4c. Replace the Widget UI

Open `OrderTrackingWidgetLiveActivity.swift` (generated by Xcode in the widget target) and **replace its entire contents** with the code from `setup/swift/OrderTrackingWidgetLiveActivity.swift`.

Delete any other auto-generated widget files (e.g., `OrderTrackingWidget.swift`, `OrderTrackingWidgetBundle.swift`) that you don't need, or leave them — only the Live Activity file matters for this feature.

## Step 5: Capabilities & Info.plist

### Main App Target

1. Select the **App** target → **Signing & Capabilities**
2. Click **+ Capability** → add:
   - ✅ **Push Notifications**
   - ✅ **Background Modes** → check **Remote notifications**

3. In `ios/App/App/Info.plist`, add these keys:

```xml
<key>NSSupportsLiveActivities</key>
<true/>
<key>NSFaceIDUsageDescription</key>
<string>Use Face ID to authenticate payments</string>
```

## Step 6: Secrets Configuration

The following secrets should already be configured in the backend. If not, add them:

### APNs Secrets (for Live Activity push updates)

| Secret | Description | Where to get it |
|--------|-------------|-----------------|
| `APNS_KEY_ID` | 10-character Key ID | [Apple Developer → Keys](https://developer.apple.com/account/resources/authkeys/list) |
| `APNS_TEAM_ID` | 10-character Team ID | [Apple Developer → Membership](https://developer.apple.com/account/#/membership/) |
| `APNS_AUTH_KEY_P8` | Full `.p8` file contents (including BEGIN/END lines) | Downloaded when creating the APNs key |

### Firebase Secret (for FCM push notifications)

| Secret | Description | Where to get it |
|--------|-------------|-----------------|
| `FIREBASE_SERVICE_ACCOUNT_JSON` | Service account JSON for FCM v2 API | Firebase Console → Project Settings → Service Accounts → Generate new private key |

### Optional

| Secret | Description |
|--------|-------------|
| `APNS_ENVIRONMENT` | `development` for testing, `production` for App Store (default: `production`) |

## Step 7: Build & Run

```bash
npm run build
npx cap sync ios
npx cap open ios
```

1. Select your **physical iOS device** in Xcode
2. Build and run (**Cmd + R**)
3. Accept the push notification permission prompt
4. Place an order → navigate to Order Tracking
5. The Live Activity appears on the Lock Screen and Dynamic Island!

## How It Works

### Push Notifications Flow

1. **App launches** → `AppDelegate.swift` initializes Firebase and requests push permission
2. **FCM token received** → stored in `UserDefaults` under key `CapacitorStorage.fcm_push_token`
3. **JS layer** reads the FCM token via `@capacitor/preferences` plugin (`Preferences.get({ key: 'fcm_push_token' })`)
4. **Token registered** → sent to the `register-push-device` edge function, saved in `push_device_tokens` table
5. **Staff changes order status** → `send-order-push` edge function sends FCM notification via HTTP v2 API
6. **Notification taps** → deep-linked to `/order-tracking/${orderId}`

### Live Activities Flow

#### Client-side
1. User opens `/order-tracking/:id` → app calls `startOrderLiveActivity()`
2. The custom `LiveActivityPlugin` starts a Live Activity with push token support
3. Plugin emits `liveActivityPushToken` event with the APNs push-to-update token
4. Token is saved to the `live_activity_tokens` database table
5. On each status change, `updateOrderLiveActivity()` updates the widget locally
6. On delivery/cancellation, `endOrderLiveActivity()` dismisses it after 5 minutes

#### Server-side
1. Staff changes order status → `send-order-push` edge function runs
2. It checks `live_activity_tokens` for the order
3. If tokens exist, sends an APNs `liveactivity` push to update the widget remotely
4. Every 2 minutes, `update-order-eta` refreshes the ETA on active Live Activities
5. On terminal status, sends `event: 'end'` with auto-dismiss

### Content State Schema

| Key | Example | Description |
|-----|---------|-------------|
| `status` | `"preparing"` | Current order status |
| `orderNumber` | `"#042"` | Display order number |
| `orderType` | `"delivery"` | delivery/pickup/dine_in |
| `statusMessage` | `"Your food is being prepared"` | Human-readable status |
| `etaMinutes` | `"12"` | Minutes until ready (empty if unknown) |
| `updatedAt` | `"2026-04-11T05:30:00Z"` | ISO timestamp of last update |

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Live Activity doesn't appear | Ensure `NSSupportsLiveActivities` is `true` in Info.plist. Must use a real device. |
| Widget shows generic content | Make sure `GenericAttributes.swift` is added to **both** the App and Widget targets. |
| Push updates don't work | Verify APNs secrets are configured. Check `APNS_ENVIRONMENT` matches your build (development vs production). |
| Token not registering | Check console for `[LiveActivity] Push token received:` and `[FCM] Token received:` logs. |
| `objectVersion 70` pod error | Run the `sed` command from Step 1 to patch the Xcode project format. |
| FCM token is 64 hex chars | That's an APNs token, not FCM. Make sure `GoogleService-Info.plist` is in the App target and Firebase is initialized. |
| Push notifications not received | Ensure Background Modes → Remote notifications is enabled. Check that `FIREBASE_SERVICE_ACCOUNT_JSON` secret is configured. |
| Pods fail to install | Run `pod install --repo-update` from `ios/App/`. Ensure CocoaPods is installed (`sudo gem install cocoapods`). |

## Quick Rebuild Cheatsheet

When you pull new changes and need to rebuild iOS:

```bash
rm -rf ios
npm install && npm run build
npx cap add ios && npx cap sync ios

# Xcode 26 fix (if needed)
sed -i '' 's/objectVersion = 70;/objectVersion = 77;/' ios/App/App.xcodeproj/project.pbxproj

# Add Firebase pods to ios/App/Podfile, then:
cd ios/App && pod install && cd ../..

npx cap open ios
```

Then in Xcode:
1. Replace `AppDelegate.swift` with the version from `setup/swift/`
2. Add `GenericAttributes.swift`, `LiveActivityPlugin.swift`, `LiveActivityPlugin.m`
3. Add Widget Extension → replace Live Activity UI with `OrderTrackingWidgetLiveActivity.swift`
4. Add `GoogleService-Info.plist`
5. Add capabilities: Push Notifications, Background Modes (Remote notifications)
6. Add `NSSupportsLiveActivities = YES` to Info.plist
7. Select device → **Cmd + R**
